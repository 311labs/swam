Templates

SWAM templates are powered by mustache.js with some extensions to improve the functionality. Templates provide an easy way to render your data into HTML.

Templates can be attached to a SWAM.View or rendered in javascript.

Templates are compiled and stored into window.template_cache based on the project folder path. For example if a template is stored in myproj/views/userinfo/userinfo.html it would be cached in window.template_cache.myproj.views.userinfo.userinfo.

See Localization for details on rendering data into different formats!

Functions

  • SWAM.renderTemplate(template, context) - render the template path using the context
  • SWAM.renderString(text, context) - render the text using the context
  • SWAM.getTemplate(path) - return the raw text for a template at the given path

Basic Syntax

{{=||| |||=}} 
<h1>{{title}}</h1>

{{#myflag}}
<p>This will only show if myflag is true</p>
{{/myflag}}
{{^myflag}}
<p>This will only show if myflag is false</p>
{{/myflag}}

<p>This will iterate through all the models in the collection</p>
<ul>
{{#collection.models}}
  <li>{{.name}} - using . will force the local context (in this case the model)</li>
{{/collection.models}}
</ul>

<p>By using 3x{ we can return raw html</p>
<div>
  {{{getIcon}}}
</div>

<p>INCLUDE Extension allows for you to include other templates with currenct context</p>
{{{INCLUDE('myapp.views.panel.example')}}}

<p>ICON Extension makes it easy to add icons</p>
{{{ICON('lock')}}}

<p>Localization Extension makes it easy to render values into different formats</p>
{{app.me.created|datetime}}

<p>You can always reference your app</p>
{{app.version}}
