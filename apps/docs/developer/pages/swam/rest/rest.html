SWAM.Rest

Remote asychronus communication is critical for any modern web application. SWAM.Rest powers SWAM.Model and SWAM.Collection, but also provides easy to use methods for direct asynchronus calls.

Authentication

When a user authenticates, the framework automatically shares these credentials with SWAM.Rest.

Option Properties

  • timeout - ms to trigger timeout (default 120000)
  • contentType - When sending data to the server, use this content type.
  • dataType - The type of data that you're expecting back from the server.

Functions

  • GET(url, data, callback, opts) - HTTP GET Request
  • POST(url, data, callback, opts) - HTTP POST Request
  • DELETE(url, data, callback, opts) - HTTP DELETE Request
  • PUT(url, data, callback, opts) - HTTP PUT Request
  • DOWNLOAD(url, data, callback, opts) - download a request to a local file

Examples

Javascript
// MAKE A REMOTE GET REQUEST
SWAM.Rest.GET("/rpc/version", null, function(resp, status) {
    if (resp.status) {
        app.api_version = resp.data;
    }
});
