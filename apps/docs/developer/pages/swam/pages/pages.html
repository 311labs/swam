SWAM.Page

Pages are how we defined routed views. A page is just a view that controls the URL in the navigation window and they can be routed directly via a URL. Pages just make organizing your web application a little easier.

  • A SWAM.Page is extended from a SWAM.View.
  • Pages have all the same properties, functions, events, as a SWAM.View with a few more added on.
  • Pages can contain many views, just like views can contain views, that contain views, and so on.
  • Pages cannot contain other pages.
  • When a model is assigned to a view it will automatically render when any changes occur.

Functions

  • updateURL(url_params) - this can be called to update the browsers url parameters
  • getRoute(params, url_params) - returns the best route for the given params for this page
  • isActivePage() - true if the page is the current active page

Callbacks

  • on_page_enter() - called when the page becomes active
  • on_page_exit() - called when the page is no longer active

Event and Actions

Pages can also handle data-action, data-showpage, data-show-url.
