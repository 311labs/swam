SWAM.View

Views create a clean object oriented interface into HTML components. Views make it simple to create and render collection of views in the DOM. Views can be bound to SWAM.Model/Collection instances for dynamic updates.

Views can be rendered from a mustache template or just the specified tagName.

When a model is assigned to a view it will automatically render when any changes occur.

Constructor Properties

  • replaces_el - should the view replace the parent or append (default false)
  • id - assign element ID
  • auto_id - automatically generate an element id
  • template - path to the mustache template
  • parent - specify the parent of this view
  • tagName - the HTML Element tag name
  • classes - the css classes to assign to the view
  • add_classes - add additional classes without changing root classes
  • model - assign a model instance to view
  • enable_swipe - support swipe/touch events on the view
  • events - bind custom events to a view
  • hal_events - bind custom hardware events to a veiw

Functions

  • on_init() - called on initialization
  • setParams(params) - set the view params
  • addChild(el_sel, view) - adds a child view
  • removeChild(el_sel) - remove the child view
  • appendChild(el_id, view) - append child without looking for element selector
  • getChild(el_sel) - returns child view
  • addToDOM($parent) - add the element to the parent jquery element
  • removeFromDOM() - remove the element from the DOM
  • render() - renders the view
  • setModel(model) - associated a model with the view
  • isInDOM() - is the view in the DOM
  • isVisible() - is the view currently visible (not necessarily by the eye)
  • isInViewport() - is visible to the eye
  • html() - returns the outer html
  • empty() - remove all appended chilren and inner HTML
  • on(event, callback, context) - subscribe events
  • off(event, callback, context) - unsubscribe events

Callbacks

Common usecase for SWAM.View is to extend the view into a new View class. The following callbacks can be overriden to handle particular events.

  • on_init() - called when a view is initialized
  • on_dom_adding() - called before adding to DOM
  • on_dom_added() - called when added to DOM
  • on_dom_removing() - called before removing from DOM
  • on_dom_removed() - called when removed from DOM
  • on_pre_render() - called before rendering
  • on_render() - called to render
  • on_post_render() - called after rendering

Event and Actions

View by default make it easy to handle custom action callbacks, showing a page, or loading an external URL. In the views HTML simply utilize data-action, data-showpage, data-show-url.
